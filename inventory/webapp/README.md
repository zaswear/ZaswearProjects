# Inventory Management WebApp – Documentation

## Overview
This repository implements a **Rust + Rocket** web application that displays an inventory of all your development projects stored in a SQLite database (`inventory.db`). The UI is rendered with **Tera** templates and runs inside WSL 2, exposing the service on `0.0.0.0:8002` (accessible from Windows via `http://127.0.0.1:8002`).

## 2️⃣ What Was Implemented
### ✅ Backend
* **Rocket 0.5‑rc.3** with `rocket_dyn_templates` (Tera) for server‑side rendering.
* **Async DB wrappers** in `db.rs` – all DB calls run inside `tokio::task::spawn_blocking` to avoid blocking the async runtime.
* **Routes** (`src/routes.rs`)
  * `GET /` → renders `index.tera` with a list of all projects.
  * `GET /project/<name>` → renders `project.tera` with full project metadata.
* Proper error handling – the index route now returns `Template` directly (no `Result`) and the project route returns `Result<Template, Status>`.

### 🎨 UI / Templates
* **`base.html`** – dark theme, Inter font, responsive table styling, subtle hover effects.
* **`index.tera`** – displays a table with columns **Nombre**, **Ruta**, **Lenguaje**, **Dependencias**. Each project name links to its detail page.
* **`project.tera`** – shows every column stored in the DB (language, dependencies, CI workflows, env vars, Docker info, runtimes, notes).

## 3️⃣ How to Build & Run
```bash
# Ensure you are inside the WSL Ubuntu environment
# (the commands below assume you are already in /home/zaswear)

# Optional: stop any stray process on the port
wsl -e bash -c "lsof -i :8002 -t | xargs -r kill -9"

# Build and launch the webapp (release mode for performance)
cd ~/inventory/webapp
RUST_BACKTRACE=1 cargo run --release
```
The console will print:
```
Rocket has launched from http://0.0.0.0:8002
```

### Access the UI
* From **Windows** browsers (Edge, Chrome, Firefox):
  `http://127.0.0.1:8002/`
* From **WSL** browsers (e.g., `w3m`):
  `http://0.0.0.0:8002/`

## 4️⃣ Generating / Updating the Inventory
The Bash script `generate_inventory.sh` walks through `~/projects/`, extracts metadata (language, dependencies, CI workflows, etc.) and stores it in `inventory.db`.
```bash
chmod +x generate_inventory.sh
./generate_inventory.sh
```
Run this whenever you add/remove projects, or set it up as a daily cron job.
